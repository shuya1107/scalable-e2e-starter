Concept (コンセプト)
3つあります！！

1つ目：ただのエゴ
「Playwrightでどこまで保守性が高く、美しい設計ができるか？」を追求した、僕の技術的な実験場（エゴ）です。

２つ目：自分がどのレベルまで設計ができるのか
単に動くテストを書くだけであれば、ベタ書きの方が早いかもしれません。しかし、長期運用を見据え保守性と拡張性をこだわり抜きオーバースペックにしていきたいと思い以下の点（※）に徹底的にこだわりました。

３つ目：AIを扱う
AIは書いてと言えばソースを書いてくれますが、指示をちゃんとしないとベタ書きなソース出てきます。レビューしながら行うこと
またオブジェクト指向をいかしてデータ情報は渡さず処理を抽象的に伝え作ってもらう今の世の中に必要そうだと思い行っています。


総称としてエゴの塊です！？


（※）
Open/Closed Principle (開放閉鎖の原則): 新しいテストケースを追加する際、既存のランナーコードを修正する必要がない。
Separation of Concerns (関心の分離): 「データ」「実行制御」「テストロジック」を明確に分離。


Features (特徴)

Design Patterns Implementation
    Strategy Pattern: テストロジックをクラスとして独立させ、共通インターフェースで実行。
    Factory Pattern: JSONの定義に基づいて、適切なStrategyクラスを動的に生成。
Data-Driven Testing (DDT)
     `testContent.json` (シナリオ定義) と users.json (テストデータ) を組み合わせることで、柔軟なテスト実行が可能。
